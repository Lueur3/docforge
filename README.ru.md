# DocForge

Графический интерфейс для [MarkItDown](https://github.com/microsoft/markitdown) и [Pandoc](https://pandoc.org/).
Конвертируйте документы между форматами без командной строки.

[English documentation](README.md)

---

## Возможности

### Вкладка MarkItDown

Конвертация любого поддерживаемого файла в Markdown с помощью библиотеки Microsoft MarkItDown.

Поддерживаемые входные форматы: Word, PDF, Excel, PowerPoint, HTML, изображения (с OCR) и другие.

- Путь к выходному файлу подставляется автоматически (та же папка, расширение `.md`)
- **Извлечение изображений** (опция, включена по умолчанию): MarkItDown встраивает картинки
  в виде base64; эта опция декодирует их в папку `<имя>_media` и заменяет ссылки на
  относительные — Markdown отображается с картинками в любом просмотрщике
- Поддержка аудио/видео через ffmpeg (опция, устанавливается в диалоге **Компоненты**)

### Вкладка Pandoc

Полноценная двунаправленная конвертация форматов документов через Pandoc.

Опции: оглавление (`--toc`), нумерация разделов (`--number-sections`),
стиль подсветки кода.

Для вывода в PDF можно выбрать движок и поля страницы (по умолчанию `2cm`, редактируется):

- **xelatex** (LaTeX) — классическая вёрстка, отлично для формул; держит кириллицу
- **Chromium** — рендерит настоящим браузером Chromium (то же семейство движка, что и
  Puppeteer в *Markdown Preview Enhanced* в VS Code), PDF выглядит «как веб-страница»;
  устанавливается в диалоге **Компоненты**

| Формат вывода | Расширение |
|---|---|
| Markdown | `.md` |
| HTML | `.html` |
| Word Document | `.docx` |
| EPUB | `.epub` |
| reStructuredText | `.rst` |
| Обычный текст | `.txt` |
| LaTeX | `.tex` |
| ODT | `.odt` |
| RTF | `.rtf` |
| PDF | `.pdf` |

- Расширение выходного файла обновляется автоматически при смене формата
- Формат входного файла определяется автоматически по расширению

### Вкладка «Изображения»

Извлекает встроенные изображения из файла (Word, PowerPoint, PDF, EPUB и др.) в выбранную
Вами папку, без создания Markdown — только сами картинки. Из PDF картинки извлекаются через
PyMuPDF, из остальных форматов — через MarkItDown.

### Общее

- Тёмная тема
- UTF-8 везде — кириллица и другие не-латинские символы работают без дополнительных настроек
- Конвертация выполняется в фоновом потоке — интерфейс не зависает
- В диалогах «Обзор» по умолчанию показываются только поддерживаемые типы файлов
- Кнопка **Компоненты** (в правом верхнем углу) — установить или проверить ffmpeg, MiKTeX
  и ядро в любой момент, не только при первом запуске
- При первом запуске окно настройки автоматически устанавливает MarkItDown и Pandoc
  и позволяет включить необязательные компоненты

## Требования

- Windows 10 или 11
- Python 3.10 или новее ([python.org](https://www.python.org/downloads/), отметьте «Add Python to PATH»)
- Интернет-соединение при первом запуске (для загрузки бинарника Pandoc)
- Для вывода в PDF дополнительно нужен MiKTeX — окно настройки может установить его за Вас

## Установка

```bash
git clone https://github.com/Lueur3/docforge.git
cd docforge
pip install -r requirements.txt
python main.py
```

Дальше запускайте приложение двойным кликом по **DocForge.bat**.
Если приложение не запускается, запустите **DocForge-debug.bat** — он покажет текст ошибки.

При первом запуске откроется окно настройки со списком всех компонентов и источников загрузки.
MarkItDown и Pandoc (обязательное ядро) устанавливаются автоматически; необязательные
компоненты — ffmpeg и MiKTeX — можно отключить тумблерами.

## Необязательные компоненты

### ffmpeg — аудио и видео

Включает конвертацию аудио и видео файлов во вкладке MarkItDown
(например, извлечение текста из `.mp3`, `.mp4`, `.wav`).

Устанавливается из диалога **Компоненты** (кнопка в правом верхнем углу) — при первом запуске
или в любой момент позже. Приложение использует [imageio-ffmpeg](https://github.com/imageio/imageio-ffmpeg)
— ручная загрузка не нужна. Если ffmpeg уже есть в системном PATH, он обнаруживается автоматически.

### MiKTeX — вывод в PDF

Нужен только для формата вывода **PDF** во вкладке Pandoc. Окно настройки устанавливает его
через `winget`; также можно установить вручную с [miktex.org](https://miktex.org/).

Приложение использует движок `xelatex` (поддерживает кириллицу) и автоматически включает
доустановку недостающих пакетов MiKTeX, поэтому первая сборка PDF подтягивает нужные
LaTeX-пакеты без запросов. Из-за этого первая конвертация в PDF может занять около минуты,
пока пакеты скачиваются.

### Chromium — PDF «как браузер»

Альтернативный движок PDF, который рендерит настоящим браузером Chromium (через Playwright,
то же семейство движка, что и Puppeteer), поэтому PDF выглядит как веб-страница — вся ширина,
современный CSS, — а не как документ LaTeX. Выбирается в выпадающем списке **PDF — движок**
на вкладке Pandoc. Устанавливается в диалоге **Компоненты**
(`pip install playwright` плюс загрузка Chromium ~150 МБ).

## Использование

**Вкладка MarkItDown**

1. Нажмите **Обзор** и выберите входной файл
2. Путь к выходному файлу подставится автоматически
3. Оставьте **Извлекать изображения** включённым, чтобы сохранить картинки рядом с Markdown,
   или отключите для чистого текста (полезно, если Markdown готовится для LLM)
4. Нажмите **Конвертировать**

**Вкладка Pandoc**

1. Нажмите **Обзор** и выберите входной файл
2. Выберите формат вывода из выпадающего списка
3. При необходимости включите оглавление, нумерацию разделов или стиль подсветки кода
4. Для PDF выберите движок и поля (активны только когда формат — PDF)
5. Путь к выходному файлу обновится автоматически — скорректируйте при необходимости
6. Нажмите **Конвертировать**

**Вкладка «Изображения»**

1. Нажмите **Обзор** и выберите файл с изображениями
2. Укажите папку назначения
3. Нажмите **Извлечь изображения**

Результаты конвертации и ошибки отображаются в области лога в нижней части каждой вкладки.

## Логи

Подробный лог пишется в `%APPDATA%\DocForge\logs\docforge.log` (с ротацией и несколькими
резервными копиями). В нём фиксируется каждая конвертация — входной файл, его формат,
целевой формат, путь вывода, опции, использованный инструмент и его версия — а также полные
трейсбеки ошибок.

Путь к файлу лога показан в строке состояния внизу окна; кликните по нему, чтобы открыть
папку. Если нужно сообщить о проблеме, приложите этот файл.

## Структура проекта

```
main.py                    лаунчер (запускается из DocForge.bat)
smoke_test.py              офлайн-самопроверка всех путей конвертации
docforge/
  app.py                   точка входа: лог, тема, зависимости, окно
  theme.py                 тёмная тема
  logging_setup.py         запись лога в файл
  proc.py                  общий флаг subprocess (без окна консоли)
  core/                    логика, без UI
    markitdown.py          файл → Markdown
    pandoc.py              форматы/опции Pandoc (данные)
    images.py              извлечение картинок (MarkItDown + PyMuPDF)
    chromium.py            HTML → PDF через Playwright/Chromium
    latex.py               поиск LaTeX-движка (MiKTeX)
    ffmpeg.py              поиск ffmpeg
    installer.py           проверка и установка зависимостей
  ui/                      интерфейс PyQt6
    window.py              главное окно
    setup_dialog.py        диалог «Компоненты»
    widgets.py             строка статуса + окно лога
    file_filters.py        фильтры диалога «Обзор»
    tabs/                  вкладки MarkItDown / Pandoc / Изображения
  resources/app.ico        иконка приложения
```
